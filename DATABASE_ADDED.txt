================================================================================
                            DATABASE SCHEMA
================================================================================

5 TABLES CREATED:

1. analyses (Main analysis records)
   - request_id (UNIQUE, indexed)
   - si_score, risk_level, latency_seconds
   - input_filename, input_file_hash (for deduplication)
   - image paths (segmentation_mask, skeleton, keypoints, etc.)
   - created_at (indexed)

2. damage_metrics (Damage breakdown)
   - analysis_id (FK to analyses)
   - density_damage, network_damage, complexity_damage, width_damage, total_damage

3. graph_features (Topology metrics)
   - analysis_id (FK to analyses)
   - total_crack_length, num_branches, num_endpoints, num_junctions
   - longest_path, graph_diameter, mean_node_degree, connectivity_score

4. post_processing_stats (Filtering statistics)
   - analysis_id (FK to analyses)
   - raw_pixels, filtered_pixels, final_pixels, filtering_applied

5. api_audit_logs (Request audit trail)
   - request_id, api_key_hash, endpoint, method, status_code
   - response_time_ms, error_message, ip_address
   - created_at (indexed)

TOTAL: 5 tables with 40+ columns and 10+ indexes

================================================================================
                          TROUBLESHOOTING
================================================================================

CONNECTION REFUSED:
  1. Check PostgreSQL is running: docker ps | grep postgres
  2. Check connection string: echo $DATABASE_URL
  3. Test connection: psql $DATABASE_URL -c "SELECT 1"

TABLES NOT CREATED:
  1. Run initialization: python -m db.init_db
  2. Check tables: psql $DATABASE_URL -c "\dt"

SLOW QUERIES:
  SELECT query, calls, mean_time FROM pg_stat_statements 
  ORDER BY mean_time DESC LIMIT 10;

CONNECTION POOL EXHAUSTED:
  1. Increase pool size in .env: DB_POOL_SIZE=20
  2. Restart API
